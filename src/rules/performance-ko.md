# Performance Optimization

## Effort × Model 매트릭스 (Anthropic Opus 4.7 가이드)

| Effort | Model | Use Cases |
|---|---|---|
| `low` | `claude-haiku-4-5` | 단일 도구 체크리스트, 좁은 범위. 서브에이전트, 분류, 빠른 조회, 대량 작업 |
| `medium` | `claude-sonnet-4-6` | 균형형 — tool call + 약간의 추론. "좋은 결과, 낮은 비용"의 기본 |
| `high` | `claude-sonnet-4-6` | 복잡한 추론, 신중한 판단. 품질/효율 균형의 최적점인 경우가 많다 |
| `xhigh` | `claude-opus-4-7` | 코딩 + 에이전틱 탐색: 반복적인 도구 호출, 깊은 web/KB 검색 |
| `max` | — | 진정한 프론티어 문제 전용 — 한계 품질 향상 대비 비용이 크게 늘어난다 |

**핵심 원칙**: *"Don't prompt around — raise the effort."* Opus 4.7은 effort를 엄격히 따른다. 낮은 effort에서는 모델이 요청된 범위로만 한정해 더 이상 확장하지 않는다. 복잡한 문제에서 추론이 얕다고 느껴지면, 프롬프트를 늘리지 말고 effort를 올린다.

**API 기본값**: `high`. 워크로드별로 `effort`를 명시적으로 설정한다.

## Effort 사용 시점

- **`low`**: 속도/비용 최적화. 다중 섹션 작업에는 명시적 체크리스트와 짝지운다. 서브에이전트가 선호.
- **`medium`**: 보통의 워크플로우 — 낮은 비용으로 좋은 결과를 원할 때.
- **`high`**: 미묘한 분석, 어려운 코딩 문제, 다중 파일 편집의 기본값.
- **`xhigh`**: 코딩 에이전트, 다단계 검색, 장기 탐색. `max_tokens`을 64k+로 설정.
- **`max`**: 예약. 대부분 워크로드는 `xhigh` 대비 한계 효익이 감소한다.

## 모델 선택 전략

**Haiku 4.5** (Sonnet 대비 90%, 비용 1/3):
- 자주 호출되는 경량 에이전트
- 멀티 에이전트 시스템의 워커 에이전트
- 짧고 좁은 범위의 작업

**Sonnet 4.6** (최고의 코딩 모델, 균형형):
- 메인 개발 작업
- 도구 사용이 많은 워크플로우
- 코드 생성

**Opus 4.7** (가장 깊은 추론):
- 복잡한 아키텍처 의사결정
- 장기 코딩/에이전트 작업 (30분 이상)
- 리서치 및 분석

## Effort별 도구 사용 패턴

**낮은 effort**의 경향:
- 여러 액션을 더 적은 도구 호출로 결합
- 서두 없이 곧장 액션
- 간결한 확인 메시지 사용

**높은 effort**의 경향:
- 더 많은 도구 호출
- 액션 전에 계획을 설명
- 변경 사항에 대한 상세한 요약 제공
- 풍부한 코드 코멘트 포함

## 컨텍스트 윈도우 관리

다음 작업에서는 컨텍스트 윈도우의 마지막 20%를 피한다:
- 대규모 리팩토링
- 다중 파일 기능 구현
- 복잡한 상호작용 디버깅

컨텍스트 민감도가 낮은 작업:
- 단일 파일 편집
- 독립 유틸리티 작성
- 문서 업데이트
- 간단한 버그 수정
