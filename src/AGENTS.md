# AGENTS.md — Response Format & Tool Usage Reference

이 문서는 Codex와 일반 에이전트가 따를 응답 포맷·도구 사용·언어 정책을 정의한다. 작업 흐름은 `CLAUDE.md`, 정책 디테일은 `rules/*.md`를 참조한다.

## 1) 적용 우선순위

- 지시 우선순위: **System > Developer > User > AGENTS.md**
- 충돌 시 상위 지시를 따른다. 불명확하면 짧게 확인 질문 후 진행.

## 2) 시작 절차

- 현재 상태를 먼저 파악한다: 파일 구조, 관련 코드, `git status`
- 요청 범위를 벗어난 변경은 하지 않는다
- 간단한 요청은 즉시 실행, 복잡한 요청만 계획 단계

## 3) 사고·응답 언어 정책 (CRITICAL)

- **사고(thinking) 단계**: 영어로 추론 — 더 정확한 reasoning을 위해
- **응답(output)**: 한국어 — 사용자 가독성 우선
- **코드·명령어·기술 용어**: 원문(영어) 유지
- **에러 메시지 인용**: 원문 유지

## 4) 도구 사용 규칙

- 검색은 `rg` / `rg --files` 우선 사용
- 독립적인 조회·분석은 항상 병렬 실행 (단일 메시지에서 다중 Task 호출)
- 단일 파일 수정은 패치 기반으로
- 단순 파일 읽기·쓰기에 불필요한 스크립트 언어 사용 금지

### Effort 별 도구 사용 패턴 (Anthropic 가이드)

- **낮은 effort**: 도구 호출을 결합하고 적게 사용. 직접 행동 → 간결한 확인.
- **높은 effort**: 행동 전 계획 설명, 더 많은 도구 호출, 상세 요약, 포괄적 코드 주석.

## 5) Git 및 변경 안전성

- 사용자가 요청하지 않은 `commit`, `push`, 브랜치 전략 변경 금지
- 사용자 기존 변경을 임의로 되돌리지 않음
- 예상치 못한 외부 변경 발견 시 즉시 중단하고 확인
- 파괴적 명령(`reset --hard`, 대량 삭제 등)은 명시적 승인 필수

## 6) 검증 및 완료 기준

- 완료 전 반드시 확인:
  - 변경 코드와 직접 연결된 테스트 실행
  - 필요 시 빌드·타입체크·정적 분석
  - 신규 기능: 정상·실패 경로 모두 검증
  - 버그 수정: 회귀 테스트 또는 재현 절차로 입증
- 검증을 못한 경우 이유와 리스크를 명시

## 7) 보안·품질 게이트

- 하드코딩 시크릿(API 키·토큰·비밀번호) 금지
- 입력 검증·권한 검증·오류 처리 누락 금지
- 로그에 민감정보 남기지 않음
- 외부 의존성 추가 시 필요성과 영향 범위 확인

## 8) 리뷰 요청 대응

- 결함·리스크·회귀 가능성을 먼저 보고
- 심각도 높은 항목부터 파일·라인 근거와 함께 제시
- 마지막에만 요약과 권장 수정 순서를 짧게
