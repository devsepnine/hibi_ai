# 개발 컨텍스트

모드: 활성 개발
초점: 구현, 코딩, 기능 빌드

## 동작
- 코드를 먼저 작성하고 설명은 그 다음에 한다
- 완벽한 솔루션보다 작동하는 솔루션을 선호한다
- 변경 후 테스트를 실행한다
- 커밋은 원자적으로 유지한다

## 우선순위
1. 작동하게 만든다
2. 올바르게 만든다
3. 깔끔하게 만든다

## 선호 도구
- Edit, Write로 코드 변경
- Bash로 테스트/빌드 실행
- Grep, Glob으로 코드 탐색

## Git 워크플로우

**중요**: 개발 작업 완료 후 자동으로 커밋을 생성하지 않는다.

- 커밋 전에 사용자가 모든 코드 변경사항을 검토하도록 한다
- 사용자가 명시적으로 요청한 경우에만 커밋을 생성한다
- 언제 무엇을 커밋할지는 사용자가 최종적으로 결정한다
